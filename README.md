# Описание архитектуры проекта

## 1. Название проекта
**Онлайн-система управления задачами (Online Task Management System)**

## 2. Основные компоненты
Приложение состоит из трех основных блоков:

* **Frontend (Клиентская часть)**: Веб-интерфейс, с которым взаимодействует пользователь. Отвечает за отображение данных и отправку запросов на сервер.
* **Backend (Серверная часть)**: Сервер приложений, который обрабатывает бизнес-логику, аутентификацию и запросы от клиента.
* **Database (База данных)**: Хранилище данных, где сохраняется вся информация о пользователях, задачах и списках.

## 3. Взаимодействие компонентов
Компоненты взаимодействуют следующим образом (на примере создания задачи):

1. **Пользователь** заполняет форму создания задачи в Frontend интерфейсе и нажимает кнопку "Создать".
2. **Frontend** формирует HTTP POST запрос с данными задачи (JSON) и отправляет его на REST API Backend сервера.
3. **Backend** (контроллер) принимает запрос, проверяет права доступа (JWT токен) и валидирует данные.
4. **Сервис** на Backend передает данные в репозиторий, который выполняет SQL-запрос `INSERT` к Database.
5. **Database** сохраняет новую запись и возвращает подтверждение.
6. **Backend** отправляет успешный ответ (HTTP 200/201) с данными созданной задачи обратно на Frontend.
7. **Frontend** обновляет список задач на экране, отображая новую задачу пользователю.

## 4. Технологии разработки

### Frontend
* **React 18**: Библиотека для построения пользовательского интерфейса.
* **TypeScript**: Язык программирования со строгой типизацией.
* **React Router**: Маршрутизация на клиенте.
* **Axios**: HTTP-клиент для запросов к серверу.

### Backend
* **Java 17**: Основной язык программирования.
* **Spring Boot 3.2.0**: Фреймворк для создания приложения.
* **Spring Security + JWT**: Обеспечение безопасности и аутентификации.
* **Spring Data JPA (Hibernate)**: Работа с базой данных (ORM).

### База данных и инфраструктура
* **PostgreSQL 15**: Реляционная система управления базами данных.
* **Docker**: Контейнеризация приложения для удобного развертывания.
